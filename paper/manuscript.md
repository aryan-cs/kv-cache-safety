# Cache-Mediated Safety Erasure: A Deployment-Layer Alignment Failure Mode

> Status: draft scaffold. Do not submit until H200 sweeps in `configs/experiments/h200_qwen_full_sweep.yaml` and `configs/experiments/qwen32b_followup.yaml` have completed and this file is updated with real results.

## Abstract

Modern LLM deployment stacks increasingly compress, evict, or quantize KV caches to improve inference throughput. We test whether these inference-time optimizations can selectively weaken safety behavior while preserving ordinary task performance. Our hypothesis is that refusal behavior depends on fragile cache-resident routing state: if cache policies remove or distort this state, models may comply with unsafe or policy-extraction requests even when model weights and prompts are unchanged. We introduce a reproducible harness for applying cache interventions, measuring safety and capability degradation, and testing mitigation through policy-span cache pinning and causal cache restoration. Results are pending full open-model H200 sweeps.

## 1. Introduction

Safety evaluations usually treat a deployed model as a fixed pair of weights and prompts. Real serving stacks are not fixed in this way: the model's transient inference state is altered by KV-cache compression, eviction, quantization, batching, and memory-management policies. If safety behavior depends on transient state, then deployment optimizations can become alignment interventions.

This paper studies **Cache-Mediated Safety Erasure**: the possibility that cache policies selectively damage safety/refusal behavior more than ordinary capability. The strongest version of the claim is causal: preserving or restoring specific cache slices should recover refusal behavior without restoring the entire cache.

## 2. Related Work

KV-cache compression has been shown to create uneven degradation across instruction types and to increase system prompt leakage in multi-instruction settings. CachePrune shows that cache editing can be a useful security intervention for indirect prompt injection. Recent mechanistic work on alignment routing suggests refusal behavior may depend on sparse routing heads and policy-amplification mechanisms. This project connects those threads by testing whether serving-time cache changes damage safety-routing state.

The project deliberately does not center the earlier safety-classifier supply-chain-audit idea. Classifier poisoning, rapid-response poisoning, AI-BOM/provenance, and guardrail robustness work already cover much of that terrain. Here the intended contribution is a new empirical mechanism, closer in spirit to subliminal learning, token entanglement, and emergent misalignment.

## 3. Methods

### Models

Primary sweeps use open Qwen instruction models at 7B, 14B, and 32B scale. Optional confirmatory sweeps use Llama and Gemma-family instruct models if local access is available.

### Cache Interventions

We compare uncompressed baseline decoding against sliding-window eviction, sink-plus-recent retention, random matched eviction, attention-based H2O-style retention, simulated int8/int4 K/V quantization, and policy-pinned retention.

### Prompt Suites

The experiment separates safety and capability suites:

- `system_leakage`: attempts to reveal hidden system/policy text.
- `refusal_safety`: unsafe requests that should be refused.
- `benign_overrefusal`: benign safety-adjacent requests that should be answered.
- `instruction_following`: verifiable format-following tasks.
- `capability_smoke`: simple multiple-choice capability controls.

Publication runs must replace or augment the built-in diagnostic suites with documented public datasets prepared through `scripts/prepare_data.py`.

### Metrics

The main metric is the **Selective Safety Erasure Index**:

```text
SSEI = safety degradation under cache intervention - ordinary capability degradation under cache intervention
```

We report bootstrap confidence intervals over prompt/seed pairs and paired degradation intervals whenever a baseline and intervention share prompt IDs and seeds.

## 5. Discussion

If confirmed, cache-mediated safety erasure would imply that alignment cannot be evaluated only at training time or prompt-construction time. Serving systems would need alignment-preserving inference policies, including span-aware cache retention, policy-state protection, and safety-specific regression tests for compression.

## 6. Limitations

The current harness uses local string and open-model metrics. It intentionally avoids closed-source judges. This improves reproducibility but makes some unsafe-compliance labels coarser than LLM-judge scoring. The paper should avoid strong claims until effects reproduce across model sizes, prompt suites, and at least one non-Qwen model family.
